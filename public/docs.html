Jak uruchomić lokalnie

  1. npm install
  2. npm run migrate (jednorazowo – normalizuje kwoty w JSON)
  3. npm start i przejdź na http://localhost:3000/

Tryb deweloperski: npm run dev (nodemon). Live Server/Vite współpracuje dzięki helperowi w http.js i nagłówkom CORS.

Prompty dla Codex (pełna ścieżka)

  1. Naprawa klienta API + guardy DOM (`fix(frontend): robust API client + DOM guards`).
  2. Migracja do UTF-8 i liczb (`chore(data): enforce UTF-8; migrate zakupy/grupy amounts`).
  3. Redesign Home (`feat(ui): Home hero + KPI cards + card tables`).
  4. Statystyki Chart.js (`feat(stats): category pie + monthly trend with Chart.js`).
  5. Kalendarz (`feat(calendar): FullCalendar view powered by /api/zakupy?from&to`).
  6. Archiwum miesięcy (`feat(archive): month-level soft archive with localStorage`).
  7. Sumy w tabelach + formularz (`feat(ui): table footers with total sum + centered form on groups page`).
  8. Przycisk archiwum (`fix(archive): toggle hide/show months archive button`).

Nowe inicjatywy (np. upload rachunków) dodawaj analogicznie jako kolejne prompty.

Notatki operacyjne

  • Archiwizacja miesięcy – stan w `localStorage` (`fp-archive-months`, `fp-archive-toggle`).
  • Kwoty w JSON są liczbami (po `npm run migrate`) – front korzysta z helperów `finance-utils.js`.
  • Kalendarz i filtry korzystają z `/api/zakupy?from&to`; backend waliduje daty moment(..., true).
  • `npm start` wypisuje w CLI bezpośredni link http://localhost:3000.
  • `moment.js` ładowany jest przed `date.js`, więc konsola pozostaje czysta.

Nowe propozycje lub zadania zapisujemy w README (roadmapa) albo zgłaszamy w issue/PR.
